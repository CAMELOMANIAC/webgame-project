# Ghost Extraction (가칭) 기획서

이 문서는 비동기 데이터 매칭 기반의 하드코어 익스트랙션 오토 배틀러 게임인 **Ghost Extraction**의 핵심 설계와 개발 로드맵을 담고 있습니다.

---

## 1. 게임 컨셉 (Game Concept)

### 1.1 핵심 요약 (Elevator Pitch)

- "과거의 기록(Ghost)이 머무는 가상 공간에서 실시간 스테미너를 관리하며 무기를 순차 발동하여 적을 격파하고, 무사히 생환하여 전리품을 챙기는 **하드코어 데이터 익스트랙션**"

### 1.2 핵심 루프 (Core Loop)

1.  **[무장]**: 창고(Stash)에서 최대 6개의 무기를 슬롯에 장착. (출격 시 분실 위험 감수)
2.  **[탐사]**: 그래프 노드 기반의 필드를 이동하며 고스트(유저 로그) 또는 NPC와 조우.
3.  **[전투]**: 실시간 틱(Tick) 기반으로 차오르는 스테미너를 소모하며 1~6번 무기를 자동 순차 사용.
4.  **[파밍]**: 처치한 고스트의 장비 데이터를 복제하여 획득. 무게 제한 내에서 전략적 선택.
5.  **[추출]**: 고정된 추출 룸에서 생환하여 획득한 아이템을 안전한 창고(Stash)로 전송.

---

## 2. 주요 시스템 상세 (System Design)

### 2.1 틱(Tick) 기반 전투 시스템

- **순차 발동**: 1번 슬롯부터 6번 슬롯까지 순서대로 체크하며, 스테미너와 쿨타임이 충족되면 즉시 무기 사용.
- **실시간 스테미너**: 턴제가 아닌 틱 단위로 자동 회복되며, 무기마다 고유의 소모값과 선/후딜레이 존재.
- **UI 피드백**: 무기 아이콘 하단의 바(Bar) 형태 게이지로 쿨다운 및 준비 상태 시각화.

### 2.2 필드 및 이동 (Field & Navigation)

- **노드 그래프**: 도시 도로와 건물이 유기적으로 연결된 구조이며, 노드 간 자유로운 이동 가능.
- **시간의 흐름 (Time)**: 룸(건물/도로) 1개 이동 시 일정 시간(예: 1시간) 경과.
- **위험도와 시간**: 탐험 시작 후 경과 시간이 길어질수록(심야로 갈수록) 조우하는 고스트의 난이도 급상승.
- **위험도 가시성**: 진입 전, 해당 위치의 고스트 밀집도 및 예상 레벨을 기반으로 위험도 사전 표시.

### 2.3 고스트(Ghost) 시스템

- **비동기 기록**: 플레이어가 각 노드를 통과할 때의 최종 상태(HP, 무기, 통과 시각 등)를 서버 DB에 저장.
- **슬라이딩 타임 매칭 (Sliding Time Matching)**:
  - 내부적으로는 **진입 시각 기준 ±15~20분 내외**의 고스트를 우선 매칭하여, 시각 차이가 너무 큰(예: 12:01 vs 12:59) 유저 간의 매칭 불균형 방지.
  - 데이터 부족 시 매칭 범위를 점진적으로 확장하는 유연한 알고리즘 적용.

### 2.4 인벤토리 및 무게 (Inventory & Weight)

- **Backpack (탐사 중)**: 격자형 인벤토리. 탐사 중 획득한 전리품 보관. 사망 시 모두 분실.
- **Stash (안전 가옥)**: 추출(Extraction)에 성공한 아이템이 보관되는 영구 저장소.
- **즉시 장착**: 필드에서 획득한 무기를 즉시 6개 슬롯 중 하나와 교체 가능.

---

## 3. 비주얼 및 UI/UX 방향성

### 3.1 "Complex but Clean" (3단 스택 구조)

- **상단 (Context Area)**: 상황에 따라 '이동 네비게이션' 혹은 '전투 로그'가 슬라이딩 방식으로 교체됨.
- **중앙 (Status Area)**: 캐릭터 상태(HP/스테미너)와 현재 장착 중인 6개의 무기 슬롯(Equipment) 고정 배치.
- **하단 (Inventory Area)**: 필요 시 `backpack` 파라미터를 통해 가방 목록을 하단에서 위로 확장.

### 3.2 애니메이션 정책 (Seamless Experience)

- **Framer Motion 적용**: 모든 레이아웃 변화는 부드러운 스프링 애니메이션으로 연결됨.
- **상태 전환**: `TanStack Router`의 Search Params를 활용하여 브라우저 히스토리와 UI 상태(탭 열림 등)를 동기화.
- **타격감**: 전투 발생 시 화면 진동 및 UI 요소의 미세한 리액션을 통해 데이터 기반 전투의 시각적 피드백 강화.

---

## 4. 개발 로드맵 (Roadmap)

### Phase 1: 핵심 시뮬레이터 및 UI 기반 구축 (진행 중)

- `motion/react` 기반의 3단 스택 레이아웃 시스템 구현.
- 무기 장착(Equipment) 및 백팩(Backpack) UI 기본 구조 설계.
- 실시간 틱 기반 스테미너 소모 및 무기 순차 발동 로직 연동.
- 전투 로그(CombatLog) 실시간 스트리밍 연출.

### Phase 2: 서버 및 데이터 연동

- 유저 상태 기록 및 고스트 매칭을 위한 DB 스키마 설계.
- 노드 기반 룸 이동 시스템 및 날짜(Day) 처리 로직.

### Phase 3: 콘텐츠 및 UI 확장

- 추출 시스템 및 무게 제한 로직 구현.
- 룸 위험도 표시 및 전투 리플레이 시스템 개발.
- 추출한 아이템을 사고 팔 수 있는 플리마켓(Market) 구현.
